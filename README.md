# نظام إدارة مصنع الإلكترونيات (Electronics Factory Management System)

نظام ويب بلغة **PHP** وقاعدة بيانات **MySQL** لإدارة مخزون منتجات مصنع إلكترونيات: إضافة المنتجات وتعديلها وحذفها وعرضها، مع تتبّع تلقائي لحالة المخزون (متوفر / منخفض / غير متوفر).

**إعداد:**بسام محمد موسى علي 1301205713
---

## نظرة عامة

يوفر النظام واجهة عربية كاملة (RTL) تغطي:

- تسجيل الدخول والخروج (حماية الوصول للنظام بجلسة Session)
- لوحة تحكم تعرض إحصائيات عامة عن المخزون
- عرض قائمة المنتجات مع البحث والتصفية حسب الحالة
- إضافة / تعديل / عرض تفاصيل / حذف (حذف ناعم) لكل منتج
- تحديث تلقائي لحالة المنتج (متوفر / منخفض / غير متوفر) بناءً على الكمية، عبر دالة PHP و Trigger في قاعدة البيانات

## التقنيات المستخدمة

| التقنية | الاستخدام |
|---|---|
| PHP | منطق الخادم (Server-Side) |
| MySQL / MySQLi | تخزين البيانات والتعامل معها (Prepared Statements) |
| HTML5 / CSS3 | واجهات المستخدم، بدعم كامل لاتجاه RTL |
| PHP Sessions | إدارة جلسة تسجيل الدخول |

## هيكلية المشروع

```
Electronics-factory-management-system/
├── config.php          # الاتصال بقاعدة البيانات + دوال مساعدة (تسجيل الدخول، تحديث الحالة، الحذف الناعم)
├── database.sql        # بنية قاعدة البيانات + بيانات أولية (60 منتجًا) + Trigger
├── login.php           # صفحة تسجيل الدخول
├── logout.php          # تسجيل الخروج
├── index.php           # لوحة التحكم / الإحصائيات
├── products.php        # قائمة المنتجات + البحث + التصفية + الحذف
├── add_product.php     # إضافة منتج جديد
├── edit_product.php    # تعديل منتج موجود
├── view_product.php    # عرض تفاصيل منتج
└── delete_product.php  # تأكيد وتنفيذ حذف منتج
```

## قاعدة البيانات

قاعدة البيانات `electronics_factory` تحتوي على جدول رئيسي `products` بالحقول التالية:

| الحقل | النوع | الوصف |
|---|---|---|
| id | INT (PK) | معرّف تلقائي الزيادة |
| name | VARCHAR(255) | اسم المنتج |
| category | VARCHAR(100) | فئة المنتج |
| specifications | TEXT | المواصفات الفنية |
| quantity | INT | الكمية المتوفرة |
| price | DECIMAL(10,2) | السعر |
| supplier | VARCHAR(255) | المورّد |
| manufacturing_date | DATE | تاريخ التصنيع |
| status | ENUM | in_stock / low_stock / out_of_stock (تلقائي) |
| deleted | TINYINT(1) | علامة الحذف الناعم |
| created_at / updated_at | TIMESTAMP | تواريخ الإنشاء والتحديث |

يحتوي `database.sql` أيضًا على `Trigger` باسم `update_product_status` يضبط حالة المنتج تلقائيًا (BEFORE UPDATE) بناءً على الكمية.

## متطلبات التشغيل

- خادم ويب يدعم PHP (مثل Apache عبر **XAMPP** أو **WAMP**)
- PHP 7.4 أو أحدث، مع تفعيل امتداد `mysqli`
- MySQL / MariaDB

## طريقة التشغيل

1. تثبيت بيئة خادم محلي مثل XAMPP وتشغيل خدمتي Apache و MySQL.
2. نسخ مجلد المشروع إلى مجلد الخادم المحلي (مثل `htdocs` في XAMPP).
3. فتح **phpMyAdmin** واستيراد ملف `database.sql` لإنشاء قاعدة البيانات `electronics_factory` وتعبئتها بالبيانات الأولية.
4. التأكد من إعدادات الاتصال بقاعدة البيانات في `config.php`:
   ```php
   define('DB_SERVER', 'localhost');
   define('DB_USERNAME', 'root');
   define('DB_PASSWORD', '');
   define('DB_NAME', 'electronics_factory');
   ```
5. فتح المتصفح على مسار المشروع، مثل: `http://localhost/Electronics-factory-management-system/login.php`
6. تسجيل الدخول باستخدام:
   - **اسم المستخدم:** `admin`
   - **كلمة المرور:** `admin123`

## ملاحظات ومجالات للتحسين

- بيانات الدخول (`admin` / `admin123`) مكتوبة مباشرة داخل الكود لأغراض توضيحية، وليست مخزّنة في قاعدة بيانات مستخدمين ولا مُشفّرة — يلزم تغيير ذلك قبل أي استخدام إنتاجي حقيقي.
- بعض المعاملات في `products.php` (مثل `filter` و `delete_id`) تُدمج مباشرة داخل استعلامات SQL دون `Prepared Statements`، بخلاف بقية الاستعلامات في المشروع، ما يستدعي معالجتها لتفادي مخاطر SQL Injection.
- لا يوجد حاليًا نظام صلاحيات متعدد المستويات (مستخدم واحد فقط بصلاحيات كاملة).
- يمكن مستقبلًا إضافة: تصدير تقارير (Excel/PDF)، سجل تدقيق للعمليات (Audit Log)، ونظام مستخدمين متعدد الأدوار.
